## Root cause (established by the orchestrator — verify, then build on it)
`/usr/local/bin/mimule-product-sentinel.py` check #4 ("Deploy consistency", lines ~236-249) compares newest mtime of `app/**/*.{tsx,ts,css}` vs newest `dist/assets/*.js` mtime; if source is >5s newer and `build_running` (= `pgrep -f mimule-team`, line 114) is false → FAIL "Frontend changes not deployed" → incident. On this host the standard workflow is: a builder agent edits `app/` for 1-3 hours (uncommitted WIP), the orchestrator verifies, commits, THEN builds+restarts. The sentinel (every 30 min) fires mid-session, the deploy at commit time clears it, the next session re-fires it: 7 distinct incidents since June 25, all auto-resolved by the idle sweep, recreated within days. The check's intent (finished changes must reach production) is right; its signal (any mtime drift) is wrong for this host's way of working.
